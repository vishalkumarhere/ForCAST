# State Policy Files

Drop one Markdown file per state into this folder. The app discovers them
automatically.

## Filename convention

Lowercase state name with hyphens for spaces. Examples:

    michigan.md
    new-hampshire.md
    west-virginia.md

Files whose names start with an underscore (like this README) are ignored.

## Suggested structure inside each file

There is no enforced structure — anything valid Markdown will render. A
useful template covering the four categories you mentioned:

```markdown
# Michigan

## Forest Action Plan
Brief description and a link to the current document.

## Best Management Practices (BMPs)
The state's forestry BMP manual(s) and where they apply.

## Taxes
Forest-related tax programs (e.g. Qualified Forest Property, Commercial
Forest), with links to state agency pages.

## Laws & Regulations
Relevant statutes and rules, with links to primary sources.

## Sources
1. Title — URL
2. Title — URL
```

## Sourcing principle

Only link to documents you can verify are real and currently accessible
(state DNR / forestry division pages, USFS state pages, state statute
portals). Do not summarize law from memory — link to the primary source
and let the user click through.
